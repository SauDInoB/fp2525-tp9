# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Quarto website for the UC "Ferramentas de Produtividade" (FP 2025/2026), TP9. Educational content in European Portuguese on R/RStudio and web publishing.

## Build

```bash
quarto render          # render whole site to docs/
quarto preview         # live preview
quarto render index.qmd  # render single file
```

`docs/` is the GitHub Pages source and is gitignored (never commit it). Deploy is automatic via `.github/workflows/publish.yml` on push to `main` (renders, then publishes to `gh-pages` branch).

## Datasets

```bash
Rscript scripts/gerar_datasets.R   # regenerate ficheiros/*.csv
```

Seed is fixed (`set.seed(20260421)`). Keep the seed on any regeneration so student outputs stay reproducible.

## Architecture

Three-page site driven by `_quarto.yml` (navbar entries map 1:1 to `.qmd` files):

| File | Role |
|---|---|
| `index.qmd` | revealjs slides (theme `simple`), 2-hour class |
| `guiao.qmd` | Step-by-step copy-paste guide |
| `exercicio.qmd` | Assignment brief (E3 component) |
| `custom.css` | Shared styles (slides + HTML pages), IBM Plex palette in `:root` (treat as immutable) |
| `ficheiros/*.csv` | Student-downloadable datasets, generated by `scripts/gerar_datasets.R` |

HTML pages use `cosmo` theme; slides use `revealjs`. All `.qmd` load `custom.css`.

## Editing rules

- European Portuguese only. Never: usuário, arquivo, tela. Always: utilizador, ficheiro, ecrã.
- No emojis anywhere.
- No excessive bold/italic (only when technically needed).
- No em-dashes (U+2014). Use commas, parentheses, or colons. CI grep enforces this.
- R code follows https://tiagojct.eu/notes/rst-bp/: RStudio projects, `here::here()`, native pipe `|>`, sections with `# ----`, `library()` in scripts, never `install.packages()` in scripts.
- Slide section headers: `# Secção {background-color="#2C3E50"}`.
- New page → add nav entry in `_quarto.yml` and create the `.qmd`.
- New exercise file → drop in `ficheiros/` (tracked in git).

See `AGENTS.md` for the original spec; keep both files in sync if rules change.
