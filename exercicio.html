Exercício · Análise dos dados e site Quarto do ensaio

Contexto

Este exercício é a componente da entrega E3 que incide sobre a análise dos dados do pré-teste do vosso cenário e a montagem do site Quarto que agrega o projecto do ensaio.

O trabalho é realizado em grupo, em continuidade com as entregas anteriores (E1: protocolo, E2: materiais).

O que entregar

Duas entregas: uma formativa a meio, e a apresentação final na TP11.

Entrega formativa (1 de Maio)

Mínimo viável. Site com a estrutura das cinco páginas, integrando o que já foi produzido até à TP8.

  1. Projecto Quarto website publicado com as páginas:
    • index.qmd: Início
    • protocolo.qmd: Protocolo
    • materiais.qmd: Materiais
    • resultados.qmd: Resultados
    • equipa.qmd: Equipa
  2. Conteúdo da apresentação da TP8 integrado no site (pode estar numa só página ou distribuído; serve de ponto de partida).
  3. Repositório Git com histórico de commits coerente.
  4. URL do site publicado acessível a partir do Moodle.

Apresentação final (13 de Maio, TP11)

O site é a apresentação. Deve conter todo o trabalho da disciplina, personalizado à medida do grupo (podem e devem alterar tema, acrescentar páginas extra, figuras, etc.).

Compilação final mínima:

  • Início: resumo do ensaio (cenário, objectivos, grupo)
  • Protocolo: protocolo completo do ensaio (versão revista da E1)
  • Materiais: CRF, formulário REDCap, protótipos da E2
  • Resultados: resultados preliminares do piloto (tabela de características basais + gráfico do outcome primário)
  • Equipa: membros, papéis, contactos
  • Declaração de uso de IA: obrigatória, independentemente de IA ter sido ou não utilizada. Declarar que ferramentas foram usadas e em que secções, ou declarar ausência de utilização.

Para além destes elementos, tudo o que considerem relevante mostrar do trabalho feito durante a disciplina.

Datasets

Cada grupo descarrega o dataset correspondente ao seu cenário:

Cenário Ficheiro Outcome primário
RespiRA ficheiros/respira.csv ACT às 12 semanas (act_semana12)
MindMove ficheiros/mindmove.csv PHQ-9 às 8 semanas (phq9_semana8)
GlucoCheck ficheiros/glucocheck.csv HbA1c aos 6 meses (hba1c_6meses)

Guardar o dataset em data/raw/ dentro do projecto RStudio. Não modificar o ficheiro original.

Guia de trabalho

Análise

Seguir o guião da aula. O script de análise deve:

  • Começar com library() para todos os pacotes necessários
  • Usar here::here() para todos os paths de ficheiros
  • Criar a tabela de características basais com gtsummary::tbl_summary(by = grupo)
  • Criar o gráfico do outcome primário com ggplot2
  • Não incluir install.packages() nem setwd()
  • Não guardar .RData (confirmar nas opções do RStudio)

Site Quarto

O site deve ter o ficheiro _quarto.yml configurado com:

project:
  type: website
  output-dir: docs

website:
  title: "[Nome do vosso ensaio]"
  navbar:
    left:
      - href: index.qmd
        text: Início
      - href: protocolo.qmd
        text: Protocolo
      - href: materiais.qmd
        text: Materiais
      - href: resultados.qmd
        text: Resultados
      - href: equipa.qmd
        text: Equipa

A tabela e o gráfico devem ser integrados em resultados.qmd com blocos de código R com echo: false.

Publicação

  1. Criar repositório público no GitHub com o nome do ensaio (ex: ensaio-htncare)
  2. Copiar o ficheiro .github/workflows/publish.yml deste site para o vosso repositório
  3. Fazer push para main
  4. Aguardar o primeiro run do workflow criar a branch gh-pages
  5. Activar GitHub Pages: Settings > Pages > Source “Deploy from a branch” · Branch gh-pages · Folder / (root)
  6. Verificar que o site fica acessível no URL do GitHub Pages

Como submeter

Submeter no Moodle:

  • Link para o repositório Git
  • Link para o site publicado

Ambos os links devem estar funcionais no momento da submissão.

Prazos

  • 1 de Maio de 2026: entrega formativa (site com estrutura + TP8)
  • 13 de Maio de 2026 (TP11): apresentação final (site completo)

Critérios de avaliação

Critério Descrição
Reprodutibilidade O código corre sem erros a partir de uma sessão limpa. Projecto RStudio, paths com here::here(), sem .RData.
Correcção da Tabela 1 Variáveis correctas, estatísticas adequadas ao tipo de variável, estratificação por grupo.
Correcção do gráfico Outcome primário representado correctamente, labels legíveis, tema limpo.
Integração no site Tabela e gráfico integrados em resultados.qmd com código R executável.
Estrutura do site As cinco páginas existem e têm conteúdo relevante da E1 e E2. Navegação funcional.
Commit history Commits com mensagens descritivas, histórico que reflecte o progresso do trabalho.

Leitura recomendada

RStudio: Best Practices for Scientific Work

Cobre projectos RStudio, estrutura de pastas, here::here(), estilo de código, e boas práticas de reprodutibilidade. Leitura obrigatória antes de começar o trabalho.

Documentação Quarto

Toda a documentação oficial está em quarto.org: tutoriais, referência completa de YAML e opções de chunks, exemplos para cada formato. As discussões no GitHub têm suporte activo da comunidade e da equipa da Posit. Consultar primeiro antes de perguntar.
